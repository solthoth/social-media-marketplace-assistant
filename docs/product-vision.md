# Product Vision

## Problem

Small resale workflows often depend on content stored directly inside a social media account. If that account is restricted, unavailable, or replaced, item photos, descriptions, prices, and publishing history can become hard to recover or reuse.

## Solution

Create a central inventory application where the seller captures each item once and then publishes listings to connected channels. The app should remain useful even when external accounts change because inventory records and listing state live in the application first.

## Core Workflow

1. Capture item photos.
2. Enter structured item details.
3. Save item as draft inventory.
4. Review and publish to one or more connected accounts.
5. Track listing status and sale status.
6. Relist or migrate unsold inventory when needed.

## Early MVP Scope

- Item create/edit flow.
- Photo upload placeholder flow.
- Inventory list with draft, listed, sold, and archived states.
- Backend API with health checks and future item endpoints.
- Integration abstraction designed for official platform APIs.
